# 🚀 Tailscale Unlimited Control 启动脚本使用指南

## 📋 概述

为了简化 **Tailscale Unlimited Control** 的启动和管理，我们提供了多种启动脚本，支持不同的操作系统和使用场景。

## 🛠️ 可用的启动脚本

### 1. **跨平台Node.js脚本** (推荐) 🌟
- **文件**: `start.js`
- **支持**: Windows, macOS, Linux
- **特点**: 功能最完整，跨平台兼容性最好

### 2. **Linux/macOS Shell脚本**
- **文件**: `start.sh`
- **支持**: Linux, macOS, WSL
- **特点**: 原生Shell体验，功能丰富

### 3. **Windows批处理脚本**
- **文件**: `start.bat`
- **支持**: Windows
- **特点**: 原生Windows CMD支持

### 4. **npm脚本** (最简单)
- **文件**: `package.json`
- **支持**: 所有平台
- **特点**: 使用npm命令，最简洁

## 🚀 快速开始

### 方式一：使用npm脚本 (推荐新手)

```bash
# 安装前端依赖
npm run setup

# 启动所有服务
npm start

# 查看服务状态
npm run status

# 停止所有服务
npm stop

# 重启服务
npm restart
```

### 方式二：使用Node.js脚本 (推荐)

```bash
# 启动所有服务
node start.js start

# 查看服务状态
node start.js status

# 停止所有服务
node start.js stop

# 重启服务
node start.js restart

# 查看日志
node start.js logs
```

### 方式三：使用Shell脚本 (Linux/macOS)

```bash
# 给脚本执行权限
chmod +x start.sh

# 启动所有服务
./start.sh start

# 查看服务状态
./start.sh status

# 停止所有服务
./start.sh stop

# 重启服务
./start.sh restart

# 查看日志
./start.sh logs
```

### 方式四：使用批处理脚本 (Windows)

```cmd
REM 启动所有服务
start.bat start

REM 查看服务状态
start.bat status

REM 停止所有服务
start.bat stop

REM 重启服务
start.bat restart

REM 查看日志
start.bat logs
```

## 📁 目录结构

启动脚本会创建以下目录结构：

```
tailscale/
├── start.js              # Node.js启动脚本
├── start.sh              # Shell启动脚本
├── start.bat             # Windows批处理脚本
├── package.json          # npm配置
├── logs/                 # 日志目录
│   ├── backend.log       # 后端日志
│   └── frontend.log      # 前端日志
├── pids/                 # 进程ID目录
│   ├── backend.pid       # 后端进程ID
│   └── frontend.pid      # 前端进程ID
├── cmd/unlimited-control/ # 后端源码
└── headscale-ui/         # 前端源码
```

## 📋 常用命令

### 启动服务
```bash
# 方式1: npm
npm start

# 方式2: Node.js
node start.js start

# 方式3: Shell (Linux/macOS)
./start.sh start

# 方式4: 批处理 (Windows)
start.bat start
```

### 查看状态
```bash
# 检查服务是否运行
npm run status
# 或
node start.js status
```

### 查看日志
```bash
# 实时查看所有日志
node start.js logs

# 查看后端日志
tail -f logs/backend.log

# 查看前端日志
tail -f logs/frontend.log
```

### 停止服务
```bash
# 优雅停止所有服务
npm stop
# 或
node start.js stop
```

### 重启服务
```bash
# 重启所有服务
npm restart
# 或
node start.js restart
```

## 🔍 故障排除

### 常见问题

#### 1. 端口被占用
```bash
# 检查端口占用
lsof -i :8080  # 检查后端端口
lsof -i :5173  # 检查前端端口

# 强制停止服务
npm stop
```

#### 2. 依赖缺失
```bash
# 检查Go安装
go version

# 检查Node.js安装
node --version

# 安装前端依赖
npm run setup
```

#### 3. 权限问题 (Linux/macOS)
```bash
# 给脚本执行权限
chmod +x start.sh

# 检查文件权限
ls -la start.sh
```

#### 4. 构建失败
```bash
# 清理并重新构建
npm run clean
npm run setup
npm start
```

### 日志分析

#### 后端日志位置
- **文件**: `logs/backend.log`
- **内容**: Go服务器启动、API请求、错误信息

#### 前端日志位置
- **文件**: `logs/frontend.log`
- **内容**: Vite开发服务器、编译信息、热重载

#### 实时日志查看
```bash
# 查看所有日志
tail -f logs/*.log

# 只看错误日志
grep -i error logs/*.log

# 按时间过滤
grep "$(date '+%Y-%m-%d')" logs/*.log
```

## 🔄 自动化部署

### 使用systemd (Linux)
```bash
# 创建服务文件
sudo nano /etc/systemd/system/tailscale-control.service

# 启用服务
sudo systemctl enable tailscale-control
sudo systemctl start tailscale-control
```

### 使用PM2 (跨平台)
```bash
# 安装PM2
npm install -g pm2

# 启动服务
pm2 start start.js --name "tailscale-control"

# 保存配置
pm2 save
pm2 startup
```
